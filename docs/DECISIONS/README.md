# Decisions

This folder stores durable architecture decision records. Use it only when a choice is likely to matter across multiple future tasks.

## When to add a decision

Add a decision record when the project chooses a long-term convention, architecture boundary, dependency strategy, API compatibility rule, migration approach, or similar policy.

Do not add a decision record for routine implementation details that are already clear from the code or git history.

## Naming

Use:

```text
NNNN-short-decision-title.md
```

Start from `docs/TEMPLATES/decision-template.md`.
